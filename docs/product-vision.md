# MelodySync 产品愿景文档

> 核心目标：让用户在使用过程中达到注意力高度集中，进入心流状态。
> 终极目标：人类逐步把可重复的事情交给 AI，把精力集中在真正需要人类判断的事情上。

---

## 一、任务层级模型

### 1.1 两种任务形态

**日常任务（Daily）**
- 今日事今日毕，每天零点自动清空
- 不会累积，保持清单干净
- 短期、临时、即兴的事情都放这里
- 例：做一顿中午饭、今天要回复某封邮件

**长期项目（Project）**
- 可以持续迭代的任务集合
- 有自己的循环机制（每日/每周定时任务）
- 有工作区目录绑定
- 例：绘画学习、理财管理、MelodySync 产品迭代

### 1.2 任务地图结构

**日常任务视图（左侧边栏 "日常任务" Tab）**
```
日常任务（根节点）
├── 今日临时任务（无归属）
│   ├── 做午饭
│   └── 回复邮件
├── 理财流水线（长期项目节点）
│   └── 本周资金记录（今日需要处理的）
├── 绘画学习（长期项目节点）
│   └── 今日读图（每日定时任务）
└── MelodySync（长期项目节点）
    └── 当前迭代任务
```

**单一长期项目视图（进入项目后）**
```
绘画学习（根节点）
├── 长期任务（AI 自动循环执行）
│   ├── 每日读图（每天看一张图）
│   └── 每周美术史（每周读1页）
├── 短期任务（定时执行一次）
│   └── 购买素描本
├── 等待任务（等待人类处理）
│   └── 等待老师回复
├── 收集箱（待整理的想法）
│   └── 想学水彩画
└── 快捷按钮（常驻功能，一键触发）
    └── 今日临摹打卡
```

### 1.3 任务状态流转路径

任务在使用过程中会沿着这条路径演进：

```
收集箱（随手记录）
  ↓ 整理归类
日常任务（手动处理）
  ↓ 发现可以持续做
归入长期项目（收集箱 bucket）
  ↓ 明确执行规律
升级为长期任务/短期任务（AI 自动执行）
  ↓ 发现是常驻功能
转化为快捷按钮（人类一键触发）
  ↓ 人类彻底交权
AI 全自动执行（人类只看结果）
```

**每个阶段的状态转换入口：**
- 收集箱 → 日常任务：移动 bucket 到"短期任务"或"等待任务"
- 日常任务 → 长期项目：右键菜单"归入长期项目" / workbench 底部"归入已有项目"
- 普通任务 → 长期任务/短期任务：workbench 底部"沉淀为长期项" / 右键"升级为定时任务"
- 普通任务 → 快捷按钮：workbench 底部"设为快捷按钮"
- 长期任务 → 快捷按钮：修改 persistent.kind 为 skill

## 二、心流保护机制

### 2.1 进入项目 = 进入专注模式

用户点击长期项目 → 界面只展示这个项目的内容：
- 左侧边栏：只显示该项目的任务
- 右侧任务地图：只显示该项目的节点
- 其他项目完全隐藏，不产生干扰
- 顶部有明显的"当前项目"标识和退出按钮

### 2.2 日常任务 = 全局视角

日常任务 Tab 是唯一能看到所有项目今日任务的地方：
- 以"日常任务"为根节点
- 各长期项目的今日任务作为子节点挂在下面
- 眼睛图标控制是否显示长期项目的任务（关闭 = 完全专注于今日任务）

### 2.3 快捷按钮（Skill）= 常驻功能

用户发现某件事是"常驻功能"（经常重复做、步骤固定）时，可以把它转化为快捷按钮：
- 出现在侧边栏底部，随时一键触发
- 触发后 AI 按预设的 runPrompt 执行
- 例：「今日临摹打卡」、「整理任务列表」、「查看今日理财行情」

## 三、每日清理与数据沉淀

### 3.1 每日清理（默认长期任务）

**触发时间**：每天零点（可配置）

**清理逻辑**：
1. 归档日常任务中所有今日任务（完成/未完成都归档）
2. 统计完成率，写柳比歇夫日记
3. 生成正反馈摘要（"今日完成 X 项，完成率 Y%"）
4. 更新 context-digest.md
5. 分析近 30 天行为模式，写入 Behavior Patterns

**存储位置**：
- 默认：`memory/worklog/{YYYY}/{MM}/{YYYY-MM-DD}.md`
- 可配置：每个长期项目可绑定自己的工作区目录
  - 绑定后，该项目的清理日记写入项目工作区
  - 例：绘画学习绑定 `/Users/kual/Documents/drawing/`，日记写入该目录

### 3.2 展示面板

**日常任务控制面板（整体视角）**
- 昨日完成情况（从 worklog 读取，正反馈）
- 各项目活跃度统计
- 长期项目进展摘要
- 行为模式推荐（"你最近经常做饭，要不要创建厨艺提升项目？"）

**单一项目控制面板（项目视角）**
- 只展示该项目的数据
- 近期产出（最近几次执行的结论）
- 任务完成趋势
- 沉淀摘要（digest）
- 循环配置（loop）
- 工作区绑定

## 四、AI 介入机制

### 4.1 被动介入：任务完成后的分类建议

用户完成一个任务后，AI 判断：
- 这个任务是否值得持续做？
- 是否应该归入某个已有长期项目？
- 是否应该创建新的长期项目？

**交互形式**：
- 方案A：聊天中出现一个卡片（"这件事要持续做吗？"）
- 方案B：任务右键菜单 → "归入长期项目" → 项目选择器（已实现）
- **推荐**：方案B为主，方案A作为 AI 主动推荐的补充

### 4.2 主动介入：从日常任务创建长期项目

**用户路径**：
1. 用户在日常任务中输入"我想学绘画"
2. AI 识别这是一个可以长期化的意图
3. AI 引导用户确认，帮助拆解子任务结构
4. 创建完成后，项目出现在长期项目 Tab

**AI 判断标准**：
- 包含"学习"、"提升"、"练习"、"坚持"等持续性词汇
- 目标明确（可以拆解成子任务）
- 用户历史行为中有相关任务

### 4.3 智能推荐：从行为模式提炼长期项目

**触发条件**：
- 用户在 30 天内多次做同类任务
- 多次使用 AI 协助同一类事情
- 日常任务中出现重复的关键词

**推荐机制**：
- AI 在每日清理时分析过去 30 天的任务模式
- 发现重复模式后，在控制面板生成推荐卡片
- 用户确认后，AI 自动拆解任务结构

## 五、人类作为 Tool/Skill（人机协作深化）

### 5.1 核心设想

随着长期项目的积累，AI 对用户的能力有了越来越清晰的认知。
可以把人类的能力也抽象成"技能（Skill）"，让 AI Agent 在发出等待任务时，主动调用人类的特定能力。

**示例**：
- 用户长期做摄影 → AI 知道用户有「构图审美」技能
- 用户长期做理财 → AI 知道用户有「投资判断」技能
- AI 在执行任务时遇到需要审美判断的环节 → 自动创建等待任务，指定调用「构图审美」

### 5.2 人类技能的积累路径

```
长期项目执行 → 产出结论（knownConclusions）
  → 每日清理归纳 → 技能摘要（skill digest）
  → 写入 memory/skills.md
  → AI 发出等待任务时可引用
```

**技能记录格式（memory/skills.md）**：
```
## 构图审美
- 来源：摄影计划长期项目，执行 47 次
- 核心能力：黄金分割、光线判断、色彩搭配
- 调用场景：需要视觉审美判断时

## 投资判断
- 来源：理财流水线，执行 23 次
- 核心能力：A股板块轮动、风险评估
- 调用场景：需要投资决策时
```

### 5.3 人类 vs AI Tool 的差异

| 维度 | AI Tool | 人类 Skill |
|------|---------|------------|
| 调用方式 | 直接执行 | 创建等待任务，等人类处理 |
| 响应时间 | 秒级 | 分钟~小时级 |
| 能力边界 | 明确 | 模糊，需要 AI 推断 |
| 积累方式 | 训练 | 从历史任务结论中归纳 |
| 触发条件 | 代码逻辑 | AI 判断"此处需要人类判断" |

### 5.4 实现路径

1. **短期**：在 constitution.md 中加入"人类技能调用"规则——当任务需要特定判断时，创建等待任务并在 runPrompt 中说明需要哪类能力
2. **中期**：每日清理时从 knownConclusions 中提炼技能摘要，写入 memory/skills.md
3. **长期**：AI 在发出等待任务时，自动从 skills.md 中匹配相关技能，在等待任务描述中引用

## 六、任务演进的完整闭环

```
用户随手记录
    ↓
收集箱（无压力积累）
    ↓ AI 整理建议
日常任务（手动处理）
    ↓ 发现规律
归入长期项目（明确归属）
    ↓ 建立循环
AI 自动执行（长期/短期任务）
    ↓ 发现常驻功能
快捷按钮（一键触发）
    ↓ 完全交权
AI 全自动（人类只看结果）

同时：
每次执行 → 积累结论 → 提炼人类技能
AI 发出等待任务时 → 调用人类特定技能
人类精力 → 集中在真正需要判断的等待任务上
```

## 七、用户路径示例

### 路径 A：新用户第一次使用

1. 打开 MelodySync → 日常任务 Tab → 空白界面
2. 提示："点击「开始任务」记录第一件事。做完了就划掉，每天零点自动清空。"
3. 用户输入"我想学绘画"
4. AI 识别持续性意图 → 提议创建长期项目
5. 用户确认 → 创建「绘画学习」长期项目（含每日读图、每周美术史等子任务）
6. 用户进入项目 → 专注模式

### 路径 B：老用户日常使用

1. 打开 MelodySync → 日常任务 Tab（眼睛关闭 = 只看今日 4 件事）
2. 处理完一件 → 划掉（done）
3. 发现某个任务可以持续做 → 右键"归入长期项目"
4. 需要专注某个项目 → 点击进入专注模式
5. 零点 → 自动清空，生成日记和正反馈

### 路径 C：任务交权给 AI

1. 用户发现"整理任务列表"每天都要做
2. 把它转化为快捷按钮 → 一键触发
3. 随着 AI 执行次数增多，用户不需要再点了
4. 升级为定时任务 → 每天自动执行
5. 用户彻底交权，只看结果

### 路径 D：人类技能被调用

1. AI 执行摄影项目的"选址分析"任务
2. AI 判断：此处需要「构图审美」能力
3. 自动创建等待任务："请用你的构图审美判断这三个备选地点"
4. 用户处理等待任务，AI 继续执行后续步骤
