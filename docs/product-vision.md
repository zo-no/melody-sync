# MelodySync 产品愿景文档

> 核心目标：让用户在使用过程中达到注意力高度集中，进入心流状态。

---

## 一、任务层级模型

### 1.1 两种任务形态

**日常任务（Daily）**
- 今日事今日毕，每天零点自动清空
- 不会累积，保持清单干净
- 短期、临时、即兴的事情都放这里
- 例：做一顿中午饭、今天要回复某封邮件

**长期项目（Project）**
- 可以持续迭代的任务集合
- 有自己的循环机制（每日/每周定时任务）
- 有工作区目录绑定
- 例：绘画学习、理财管理、MelodySync 产品迭代

### 1.2 任务地图结构

**日常任务视图（左侧边栏 "日常任务" Tab）**
```
日常任务（根节点）
├── 今日临时任务（无归属）
│   ├── 做午饭
│   └── 回复邮件
├── 理财流水线（长期项目节点）
│   └── 本周资金记录（今日需要处理的）
├── 绘画学习（长期项目节点）
│   └── 今日读图（每日定时任务）
└── MelodySync（长期项目节点）
    └── 当前迭代任务
```

**单一长期项目视图（进入项目后）**
```
绘画学习（根节点）
├── 长期任务
│   ├── 每日读图（每天看一张图）
│   └── 每周美术史（每周读1页）
├── 短期任务
│   └── 购买素描本
├── 等待任务
│   └── 等待老师回复
└── 收集箱
    └── 想学水彩画
```

## 二、心流保护机制

### 2.1 进入项目 = 进入专注模式

用户点击长期项目 → 界面只展示这个项目的内容：
- 左侧边栏：只显示该项目的任务
- 右侧任务地图：只显示该项目的节点
- 其他项目完全隐藏，不产生干扰
- 顶部有明显的"当前项目"标识和退出按钮

### 2.2 日常任务 = 全局视角

日常任务 Tab 是唯一能看到所有项目今日任务的地方：
- 以"日常任务"为根节点
- 各长期项目的今日任务作为子节点挂在下面
- 眼睛图标控制是否显示长期项目的任务

## 三、每日清理与数据沉淀

### 3.1 每日清理（默认长期任务）

**触发时间**：每天零点（可配置）

**清理逻辑**：
1. 归档日常任务中所有今日任务（完成/未完成都归档）
2. 统计完成率，写柳比歇夫日记
3. 生成正反馈摘要（"今日完成 X 项，完成率 Y%"）
4. 更新 context-digest.md

**存储位置**：
- 默认：`memory/worklog/{YYYY}/{MM}/{YYYY-MM-DD}.md`
- 可配置：每个长期项目可绑定自己的工作区目录
  - 绑定后，该项目的清理日记写入项目工作区
  - 例：绘画学习绑定 `/Users/kual/Documents/drawing/`，日记写入该目录

### 3.2 展示面板

**日常任务控制面板（整体视角）**
- 展示整个 MelodySync 使用过程中的项目结论汇总
- 昨日完成情况（从 worklog 读取）
- 各项目活跃度统计
- 长期项目进展摘要

**单一项目控制面板（项目视角）**
- 只展示该项目的数据
- 近期产出（最近几次执行的结论）
- 任务完成趋势
- 沉淀摘要（digest）
- 循环配置（loop）

## 四、AI 介入机制

### 4.1 被动介入：任务完成后的分类建议

用户完成一个任务后，AI 判断：
- 这个任务是否值得持续做？
- 是否应该归入某个已有长期项目？
- 是否应该创建新的长期项目？

**触发条件**：
- 任务 workflowState 变为 done
- taskCard 中有明确的 goal 或 knownConclusions
- 该任务没有归属于任何长期项目

**交互形式**：
- 方案A：聊天中出现一个卡片（"这件事要持续做吗？"）
- 方案B：任务右键菜单 → "归入长期项目" → 项目选择器（已实现）
- **推荐**：方案B为主，方案A作为 AI 主动推荐的补充

### 4.2 主动介入：从日常任务创建长期项目

**用户路径**：
1. 用户在日常任务中输入"我想学绘画"
2. AI 识别这是一个可以长期化的意图
3. AI 引导用户：
   - 确认是否要创建长期项目
   - 帮助拆解子任务（每日读图、每周美术史、技法练习）
   - 设置执行频率和触发时间
   - 绑定工作区目录（可选）
4. 创建完成后，项目出现在长期项目 Tab

**AI 判断标准**：
- 包含"学习"、"提升"、"练习"、"坚持"等持续性词汇
- 目标明确（可以拆解成子任务）
- 用户历史行为中有相关任务（见 4.3）

### 4.3 智能推荐：从行为模式提炼长期项目

**触发条件**：
- 用户在 30 天内多次做同类任务（例：多次"做饭"相关任务）
- 多次使用 AI 协助同一类事情
- 日常任务中出现重复的关键词

**推荐机制**：
- AI 在每日清理时分析过去 30 天的任务模式
- 发现重复模式后，在控制面板生成推荐卡片
- 推荐内容："你最近经常做饭，要不要创建「厨艺提升」长期项目？"
- 用户确认后，AI 自动拆解任务结构

## 五、用户路径示例

### 路径 A：新用户第一次使用

1. 打开 MelodySync → 日常任务 Tab → 空白界面
2. 提示："点击「开始任务」记录第一件事。做完了就划掉，每天零点自动清空。"
3. 用户输入"我想学绘画"
4. AI 识别持续性意图 → 弹出引导卡片
5. 用户确认 → 创建「绘画学习」长期项目
6. 项目自动包含：每日读图、每周美术史等子任务
7. 用户进入项目 → 专注模式

### 路径 B：老用户日常使用

1. 打开 MelodySync → 日常任务 Tab
2. 看到今日需要处理的任务（来自各长期项目 + 临时任务）
3. 处理完一件 → 划掉（done）
4. 发现某个任务属于某个项目 → 归入长期项目
5. 需要专注某个项目 → 点击进入专注模式
6. 零点 → 自动清空，生成日记和正反馈

### 路径 C：长期项目自迭代

1. 每日/每周定时任务自动触发
2. AI 执行任务，生成产出
3. 产出写入 digest（knownConclusions、keyPoints）
4. 下次执行时自动带入上次结论
5. 控制面板展示项目进展曲线
