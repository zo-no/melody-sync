# 任务时间线（轴）— 需求文档

> 更新：2026-03-31

---

## 一、定位

**轴**是一个全屏流程图视图，展示一个主线任务的完整生命周期。

它的目标是让用户和 AI 都能一眼看清：
- 主线走过哪些步骤（消息节点）
- 在哪里开了支线、支线做了什么
- 支线收束后带回了什么
- 接下来计划做什么

入口：头部"轴"按钮，点击打开全屏覆层。

---

## 二、视图结构

```
┌─────────────────────────────────────────────────────────┐
│ [任务目标]                          任务时间线    ✕      │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   ●─── [主线消息 1]                                     │
│   │                                                     │
│   ├──→ [支线A]  ──→ [支线A子支线1]                      │
│   │    已收束        未开始                             │
│   │         ╰──────────────────→ (回流箭头)             │
│   │                                                     │
│   ●─── [主线消息 2]  ←─ (收束回流到这里)                │
│   │                                                     │
│   ├──→ [支线B]                                          │
│   │    进行中                                           │
│   │                                                     │
│   ●─── [主线消息 3]  ← 当前（高亮）                     │
│   │                                                     │
│   ○─── [计划节点 1]  (虚线，来自 candidateBranches)     │
│   ○─── [计划节点 2]                                     │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 三、节点类型

### 3.1 主线消息节点（commit）
- 来源：主线 session 的 user message 事件
- 显示：时间戳 + 消息预览（截断到 34 字符）
- 样式：与任务地图节点一致（`quest-task-flow-node` 同款）
- 交互：点击跳转到对应消息并关闭视图
- 当前最后一条：高亮（accent 色 + 左侧竖线）

### 3.2 支线节点（branch）
- 来源：`branchContext`，`parentSessionId === rootSessionId`
- 显示：状态徽章 + 时间 + 支线目标（goal）+ 摘要（消息数 or 已收束内容）
- 样式：与主线节点同款，但有状态区分
  - 进行中：正常样式
  - 已收束（merged）：opacity 0.72 + 标题删除线
  - 已挂起（parked）：opacity 0.72 + 标题删除线
- 交互：点击切换到该支线 session
- 位置：从分叉的主线消息节点横向展开（右侧）

### 3.3 子支线节点（sub-branch）
- 来源：`branchContext`，`parentSessionId === 某支线sessionId`
- 显示：同支线节点
- 位置：从父支线节点继续横向展开（更右侧）
- 最多展示 2 层深度（主线 → 支线 → 子支线）

### 3.4 计划节点（candidate）
- 来源：主线 session `taskCard.candidateBranches`
- 显示："计划"徽章 + 标题
- 样式：虚线边框，低透明度
- 位置：主线末尾，延续主线竖线
- 交互：无（仅展示）

## 四、连线规则

| 连线类型 | 起点 | 终点 | 样式 |
|---------|------|------|------|
| 主线竖线 | 第一个主线节点 | 最后一个节点 | 实线，`border-strong` 色 |
| 分叉线 | 主线节点中心右侧 | 支线节点左侧 | 实线，弧形曲线 |
| 子分叉线 | 支线节点右侧 | 子支线左侧 | 实线，弧形曲线 |
| 收束回流线 | 支线节点右侧 | 下一个主线节点左侧 | 虚线，accent 色 |
| 计划延伸线 | 最后主线节点 | 第一个计划节点 | 虚线，`border` 色 |

**关键**：连线必须从节点的准确中心点出发/到达，不能悬空。

## 五、布局算法

```
主线节点：固定在左侧列（x = MAIN_X），从上到下排列
支线节点：从分叉主线节点向右展开（x = MAIN_X + NODE_W + BRANCH_X_GAP）
子支线：继续向右（x += NODE_W + SUB_X_GAP）

每行高度 = max(NODE_H + GAP_Y, 该行所有支线总高度 + GAP_Y)
支线总高度 = Σ(每个支线高度 + BRANCH_Y_GAP)
支线高度 = max(NODE_H, 子支线总高度)

主线节点的 Y = 累计前面所有行的高度
支线节点的 Y = 同行主线节点的 Y（第一个支线与主线节点顶部对齐）
```

## 六、当前问题（2026-03-31）

1. **连线断裂**：主线节点和支线节点之间没有连线，或连线位置错误
2. **主线只有一个节点**：布局显示只有一条主线消息，其他消息丢失
3. **支线节点位置偏移**：支线节点没有对齐到分叉的主线节点
4. **收束回流箭头缺失**：已收束支线没有回流箭头指向下一个主线节点
5. **节点高度不固定**：内容多时节点高度膨胀，破坏布局

**根本原因**：
- `NODE_H` 是估算值（72px），但实际节点高度由 CSS grid 决定，内容多时会超出
- 布局计算用的是估算高度，实际渲染高度不一致，导致节点重叠或间距错误
- 连线坐标基于估算高度计算，与实际节点位置不符

**修复方向**：
- 节点内容精简，固定为 2-3 行（badge + title + sub），用 CSS 强制固定高度
- 布局完成后用 `getBoundingClientRect()` 读取实际坐标再画连线（两步渲染）

## 七、数据来源

| 数据 | API / 字段 |
|------|-----------|
| 主线消息 | `GET /api/workbench/sessions/{id}/commit-tree` → `data.commits[]` |
| 支线节点 | 同上 → `commit.branches[]`（按 forkAtSeq 挂在对应 commit 上） |
| 子支线 | 同上 → `branch.subBranches[]` |
| 游离支线 | 同上 → `data.danglingBranches[]` |
| 计划节点 | 同上 → `data.candidates[]` |
| 任务目标 | 同上 → `data.goal` |
