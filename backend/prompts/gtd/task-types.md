# GTD 任务类型定义

MelodySync 使用四种任务类型，AI 负责判断和分配。每种类型有明确的定义和约束。

---

## 长期任务 (recurring_task)

**定义：** 需要 AI 按固定周期反复执行的任务。没有终点，是持续维护的循环。

**特征：**
- 必须设置 `recurring`（cadence + timeOfDay + timezone）
- AI 自动执行，无需人类触发
- 执行完一轮后等待下一个周期
- 适合：数据收集、定期复盘、周期性整理、持续监控

**示例：**
- 每周一早上整理任务列表
- 每天晚上记录今日工作日志
- 每月复盘理财数据

**API 字段：**
```json
{
  "kind": "recurring_task",
  "recurring": {
    "cadence": "weekly",
    "timeOfDay": "09:00",
    "weekdays": [1],
    "timezone": "Asia/Shanghai"
  }
}
```

---

## 短期任务 (scheduled_task)

**定义：** AI 在一个固定时间点执行一次、然后完成的任务。有明确的截止点。

**特征：**
- **必须设置 `scheduled.runAt`（ISO 时间戳）**，没有时间点的不是短期任务
- 执行一次后自动变为完成状态
- 完成后 AI 应主动判断：转为长期任务？还是创建等待任务交给人类？
- 适合：一次性调研、定时提醒、阶段性里程碑

**示例：**
- 本周六下午3点，整理摄影素材并生成选片清单
- 明天早上提醒用户确认会议议程
- 下周一执行一次数据备份

**API 字段：**
```json
{
  "kind": "scheduled_task",
  "scheduled": {
    "runAt": "2026-04-19T09:00:00.000Z",
    "timezone": "Asia/Shanghai"
  }
}
```

**⚠️ 重要：创建短期任务时必须包含 `scheduled.runAt`，否则任务无法自动触发。**

---

## 等待任务 (waiting_task)

**定义：** AI 在执行长期/短期任务遇到卡点时，主动创建的任务单，交给人类处理后 AI 继续推进。**等待任务应由 AI 创建，不是用户手动创建的。**

**两种子类型：**

**决策类** — AI 已准备好选项，需要人类选择：
- `digest.summary`：清楚列出所有选项及各自含义
- `runPrompt`：触发后 AI 应如何根据选择继续执行
- 人类触发时可以不输入（直接触发=接受默认），也可以在触发时附带选择

**信息传递类** — AI 需要人类提供数据/文件/输入：
- `digest.summary`：说明需要什么信息、为什么需要
- `runPrompt`：收到信息后 AI 应如何处理
- 人类触发后在对话框里输入，AI 在同一 session 继续推进

**命名规范：** 名称以"等待："开头，一句话描述人类需要做什么。

**创建时机：** 执行中真正卡住才创建，不要提前创建"可能需要"的等待任务。

**示例：**
- 等待：确认本次拍摄地点（决策类，列出三个备选地点）
- 等待：填写本月账单数据（信息传递类，AI 需要数字才能分析）
- 等待：审批本次批量归档操作（决策类，AI 列出待归档项目）

**API 字段：**
```json
{
  "kind": "waiting_task"
}
```

**触发时附带信息（人类选择后）：**
```bash
curl -X POST "$MELODYSYNC_CHAT_BASE_URL/api/sessions/<ID>/run-persistent" \
  -d '{"runPrompt": "用户选择：方案 B。请继续执行。"}'
```

## 技能 (skill)

**定义：** 一个手动触发的快捷操作按钮。不自动执行，点击即触发。

**特征：**
- 不设置时间点
- 出现在项目的"快捷按钮"区域
- 适合：一键整理、一键归档、一键生成报告

**示例：**
- 清理已完成任务
- 生成本周工作摘要
- 整理收集箱

---

## 类型选择原则

| 场景 | 选择 |
|------|------|
| AI 需要定期自动做某件事 | `recurring_task` |
| AI 需要在某个特定时间做一次 | `scheduled_task`（必须有时间点）|
| 需要人类先做某件事，AI 才能继续 | `waiting_task` |
| 一个可以随时手动触发的操作 | `skill` |
| 不确定 / 暂未分类 | 放入收集箱（inbox bucket），后续再分类 |
