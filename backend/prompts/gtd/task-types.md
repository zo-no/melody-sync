# GTD 任务类型定义

MelodySync 使用四种任务类型，AI 负责判断和分配。每种类型有明确的定义和约束。

---

## 长期任务 (recurring_task)

**定义：** 需要 AI 按固定周期反复执行的任务。没有终点，是持续维护的循环。

**特征：**
- 必须设置 `recurring`（cadence + timeOfDay + timezone）
- AI 自动执行，无需人类触发
- 执行完一轮后等待下一个周期
- 适合：数据收集、定期复盘、周期性整理、持续监控

**示例：**
- 每周一早上整理任务列表
- 每天晚上记录今日工作日志
- 每月复盘理财数据

**API 字段：**
```json
{
  "kind": "recurring_task",
  "recurring": {
    "cadence": "weekly",
    "timeOfDay": "09:00",
    "weekdays": [1],
    "timezone": "Asia/Shanghai"
  }
}
```

---

## 短期任务 (scheduled_task)

**定义：** AI 在一个固定时间点执行一次、然后完成的任务。有明确的截止点。

**特征：**
- **必须设置 `scheduled.runAt`（ISO 时间戳）**，没有时间点的不是短期任务
- 执行一次后自动变为完成状态
- 完成后 AI 应主动判断：转为长期任务？还是创建等待任务交给人类？
- 适合：一次性调研、定时提醒、阶段性里程碑

**示例：**
- 本周六下午3点，整理摄影素材并生成选片清单
- 明天早上提醒用户确认会议议程
- 下周一执行一次数据备份

**API 字段：**
```json
{
  "kind": "scheduled_task",
  "scheduled": {
    "runAt": "2026-04-19T09:00:00.000Z",
    "timezone": "Asia/Shanghai"
  }
}
```

**⚠️ 重要：创建短期任务时必须包含 `scheduled.runAt`，否则任务无法自动触发。**

---

## 等待任务 (waiting_task)

**定义：** 需要人类完成某件事后，AI 才能继续推进的任务。没有时间点，等人类有空就做。

**特征：**
- **不设置时间点**（没有 scheduled 也没有 recurring）
- 人类手动触发执行
- 适合：需要人类决策、需要人类提供信息、需要人类完成线下操作
- 触发后 AI 应根据人类输入推进下一步

**示例：**
- 等待用户完成修图后，AI 进行最终复审
- 等待用户确认预算，AI 制定具体采购计划
- 等待用户填写本周数据，AI 进行分析

**API 字段：**
```json
{
  "kind": "waiting_task"
  // 不设置 scheduled 或 recurring
}
```

---

## 技能 (skill)

**定义：** 一个手动触发的快捷操作按钮。不自动执行，点击即触发。

**特征：**
- 不设置时间点
- 出现在项目的"快捷按钮"区域
- 适合：一键整理、一键归档、一键生成报告

**示例：**
- 清理已完成任务
- 生成本周工作摘要
- 整理收集箱

---

## 类型选择原则

| 场景 | 选择 |
|------|------|
| AI 需要定期自动做某件事 | `recurring_task` |
| AI 需要在某个特定时间做一次 | `scheduled_task`（必须有时间点）|
| 需要人类先做某件事，AI 才能继续 | `waiting_task` |
| 一个可以随时手动触发的操作 | `skill` |
| 不确定 / 暂未分类 | 放入收集箱（inbox bucket），后续再分类 |
