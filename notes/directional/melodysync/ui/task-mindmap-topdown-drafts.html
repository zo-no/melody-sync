从上往下看的任务脑图稿

这三版都遵守同一条交互前提：默认只展开当前路径，其他分支折叠。区别在于“当前路径、旁支、历史节点”三者的视觉分工怎么做。

推荐 稳定性优先

方案 A · Spine

主线固定在中轴，从上往下只展开当前链，旁支折成芯片或托盘。最像“任务脑图”，同时最不容易跳。

信息密度高 对比 siblings

方案 B · Layers

每一层单独成一条横向泳道。优点是便于比较同层分支，缺点是会更像“分层看板”，脑图感弱一些。

总览优先 经典脑图感

方案 C · Classic Tree

最像传统思维导图：根节点在上，直接子节点横向排开，再在某一支下继续往下。视觉上最像“树”，但密度高时容易乱。
